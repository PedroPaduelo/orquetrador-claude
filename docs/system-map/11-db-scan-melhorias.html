Workflow Update sem $transaction — 7 operações desprotegidas

workflows.repository.ts:226-293

AS-IS (Atual)

7 operações sequenciais sem transação. Se o delete de hooks falhar, steps antigos já foram removidos mas novos não criados — workflow corrompido.

// ❌ Sem $transaction — inconsistência garantida em falha
const existingSteps = await prisma.workflowStep.findMany(...)

await prisma.workflowStepAgent.deleteMany(...)

await prisma.workflowStep.deleteMany(...)

TO-BE (Proposto)

Envolver tudo em $transaction. Se qualquer operação falhar, todas são revertidas. Workflow mantém integridade.

// ✅ Tudo atômico — falha reverte tudo
const workflow = await prisma.$transaction(async (tx) => {
  await tx.workflowStepMcpServer.deleteMany(...)
  await tx.workflowStepSkill.deleteMany(...)
  await tx.workflowStepAgent.deleteMany(...)
  await tx.workflowStepRule.deleteMany(...)
  await tx.workflowStepHook.deleteMany(...)
  await tx.workflowStep.deleteMany(...)
  return tx.workflow.update(...)
})

Nenhum list endpoint tem paginação — 10+ repositories afetados

conversations, skills, agents, rules, hooks, webhooks, workflows, messages, admin/users

AS-IS

Todos os findAll retornam tudo sem take/skip. Com crescimento do sistema, resposta fica cada vez mais lenta.

// ❌ Retorna TUDO — 10.000 conversas? Sem problema... 💀
async findAll(userId: string) {
  return prisma.conversation.findMany({
    where: { userId },
    orderBy: { createdAt: 'desc' },
    // Sem take, sem skip, sem cursor
  })
}
TO-BE

Paginação cursor-based (mais eficiente que offset) com retorno de metadados. Padrão reutilizável em todos os repositories.

// ✅ Cursor-based pagination — constante em qualquer volume
async findAll(userId: string, opts?: {
  take?: number, cursor?: string, search?: string
}) {
  const take = opts?.take ?? 20
  const [items, total] = await prisma.$transaction([
    prisma.conversation.findMany({
      where: { userId },
      take: take + 1,  // +1 para saber se tem próxima
      ...(opts?.cursor && { cursor: { id: opts.cursor }, skip: 1 }),
      orderBy: { createdAt: 'desc' },
      select: { id: true, title: true, createdAt: true },
    }),
    prisma.conversation.count({ where: { userId } }),
  ])
  return {
    items: items.slice(0, take),
    hasMore: items.length > take,
    total,
  }
}
