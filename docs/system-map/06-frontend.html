SSE Streaming (Tempo Real)
⚡
use-sse-stream.ts
Hook principal que gerencia a conexao SSE com o backend. Recebe a URL de streaming e retorna estado reativo com fase atual, conteudo e controles.
const { phase, content, actions, isStreaming, isPaused } = useSSEStream({ url: `/api/executions/${id}/stream`, onEvent: (event) => handleEvent(event), autoReconnect: true, })
🔄
Funcionalidades
✓ Auto-reconnect on page refresh (watchExecution)
✓ Interrupt/cancel support via API
✓ Pause/resume streaming
✓ Step status tracking em tempo real
✓ Error recovery com retry
✓ 16 tipos de eventos SSE processados

16 Eventos SSE Processados
Evento Descricao Acao no Frontend
execution:started Execucao iniciou Define fase = connecting, mostra timeline
execution:progress Progresso geral Atualiza barra de progresso
execution:completed Todas as steps finalizaram Define fase = idle, mostra resultado final
execution:failed Execucao falhou Mostra erro, habilita retry
execution:cancelled Usuario cancelou Limpa streaming, volta ao idle
step:started Step individual iniciou Marca step como "running" na timeline
step:progress Progresso do step Atualiza % do step individual
step:completed Step finalizou com sucesso Marca step como "done", mostra output
step:failed Step falhou Marca step como "failed" com erro
step:skipped Step pulado (condicao nao atendida) Marca step como "skipped" na timeline
stream:chunk Chunk de texto do Claude Appenda texto ao conteudo de streaming
stream:action Acao executada pelo Claude Adiciona acao a lista de acoes
stream:thinking Claude processando Define fase = ai_thinking
token:usage Consumo de tokens Atualiza metricas de custo
heartbeat Keep-alive da conexao Reseta timeout de desconexao
error Erro generico Mostra toast de erro, tenta reconectar

API Integration
📥
Axios Instance
Instancia centralizada com base URL do backend, interceptors de autenticacao (inject token no header) e interceptor de resposta para tratamento de erros.
const api = axios.create({ baseURL: API_URL, headers: { 'Content-Type': 'application/json' } }) // Request interceptor api.interceptors.request.use((config) => { const token = localStorage.getItem('token') if (token) config.headers.Authorization = `Bearer ${token}` return config })

Workflow Wizard

Formulario de 3 fases para criacao e edicao de workflows. Cada fase valida antes de avancar. O estado e mantido no Workflows Store (Zustand) para persistir entre navegacoes.

1
Basic Info
Nome, descricao, tags, agente padrao, modelo de IA, prioridade e configuracao geral do workflow.
Nome Descricao Tags Agent Model Priority
2
Step Builder
Construtor visual de steps com drag-and-drop. Cada step define: prompt, agent, skills, rules, timeout e dependencias.
Add Step Reorder Config Dependencies Templates
3
Review
Revisao final com preview visual do fluxo, validacao de dependencias, estimativa de tokens e confirmacao.
Flow Preview Validation Token Est. Save
👁
Visual Flow Preview
Na fase de Review, o wizard renderiza um preview visual do workflow com:
✓ Nodes representando cada step
✓ Setas de dependencia entre steps
✓ Cores por status (pending, ready)
✓ Tooltip com detalhes do step
✓ Deteccao de ciclos/erros
