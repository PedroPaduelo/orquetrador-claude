Core — Usuários e Workflows

User

Usuário do sistema. Owner de todos os recursos.

RBAC 12 relações
idString @id uuid
emailString @unique
passwordHashString
nameString?
githubTokenString?
roleString (admin|developer|viewer)
createdAtDateTime
updatedAtDateTime
Has many: workflows, conversations, mcpServers, skills, agents, rules, plugins, hooks, apiKeys, webhooks, promptVersions, gitAccounts

Workflow

Workflow com tipo sequential, dag ou step_by_step.

idString @id uuid
nameString
descriptionString?
typeString (sequential|dag|step_by_step)
userIdString FK → User
Has many: steps, conversations
Belongs to: User

WorkflowStep

Step individual com prompts, conditions, validators, modelo e dependências DAG.

15+ campos 5 joins
idString @id uuid
workflowIdFK → Workflow
nameString
baseUrlString
stepOrderInt
systemPromptString?
useBasePromptBoolean (true)
conditionsString JSON
maxRetriesInt (0)
dependsOnString JSON[] (DAG)
validatorsString JSON[]
backendString (claude)
modelString?
outputVariablesString JSON[]
inputVariablesString JSON[]
Belongs to: Workflow
Has many: messages, sessions, memories, logs
Join tables: McpServers, Skills, Agents, Rules, Hooks

Conversas e Mensagens

Conversation

Sessão de chat vinculada a User + Workflow. Rastreia step atual.

id, title, projectPathString
userIdFK → User
workflowIdFK → Workflow
currentStepIdFK? → WorkflowStep
Has many: messages, sessions, stepMemories, executionStates, executionLogs, executionTraces

Message

Mensagem user/assistant com metadata JSON e attachments.

roleString (user|assistant)
contentString
selectedForContextBoolean
metadataString? JSON
conversationId, stepIdFK
Has many: attachments (MessageAttachment)
Índices: conversationId, conversationId+stepId, stepId

MessageAttachment

Arquivo anexo (imagem). PNG/JPG/GIF/WebP até 10MB.

filename, mimeTypeString
sizeInt (bytes)
path, projectPath, urlString
messageIdFK → Message

ConversationSession

Resume token do Claude CLI por conversation+step.

claudeSessionIdString
conversationIdFK → Conversation
stepIdFK → WorkflowStep
Unique: conversationId + stepId

StepMemory

Resumo acumulado da conversa para context compaction.

contentString (resumo)
conversationId + stepIdFK unique

Execução e Monitoramento

ExecutionState

State machine: queued → running → completed/failed/cancelled.

idString @id (executionId)
stateString (queued|running|...)
currentStepIndexInt
retryCountsString JSON
metadataString JSON
conversationIdFK → Conversation

ExecutionLog

Audit trail de eventos por execução.

executionIdString
eventTypeString
stepId, stepNameString?
dataString JSON
Índices: executionId, executionId+eventType, conversationId

ExecutionTrace

Trace completo do Claude CLI: tokens, custo, timing, raw I/O.

40+ campos
executionId, conversationId, stepIdString
commandLineString
pidInt?
stdoutRaw, stderrRawString
exitCode, signal, resultStatusResult
inputTokens, outputTokensInt
totalCostUsdFloat?
claudeCodeVersionString?
startedAt...completedAtDateTime

StepTemplate

Template reutilizável de step com resources pré-configurados.

name, descriptionString
systemPrompt, conditionsString
resourceIdsJSON {mcpServerIds, skillIds...}

PromptVersion

Histórico de versões de prompts por step.

stepIdString
versionInt
content, diffString
createdByFK → User

UserTokenBudget

Controle de budget diário/mensal de tokens por usuário.

dailyLimitInt (5M)
monthlyLimitInt (100M)
currentDailyUsageInt
currentMonthlyUsageInt
userIdFK @unique → User

AI Config — Skills, Agents & Rules

Skill

Habilidade com frontmatter, body, allowed tools. Importável do GitHub.

name, descriptionString
frontmatter, bodyString
allowedToolsJSON[]
modelString?
sourcemanual | imported
repoOwner/Name/Branch/PathGitHub tracking
fileManifestJSON [{path,content}]

Agent

Agente com system prompt, tools, model, permissões. Importável do GitHub.

systemPromptString
tools, disallowedToolsJSON[]
permissionModeString (default)
maxTurnsInt?
skillsJSON[]

Rule

Regra (diretriz) injetada no system prompt. Pode estar vinculada a Skill.

name, description, bodyString
isGlobalBoolean
skillIdFK? → Skill
pluginIdFK? → Plugin

Integrações — MCP, Plugins, Webhooks

McpServer

Servidor MCP (http/sse/stdio) com tools cache e health check.

typehttp | sse | stdio
uri, command, argsConnection info
envVarsJSON {}
toolsCacheString? JSON
lastTestAt, lastTestOkHealth check

Plugin

Plugin agrupando MCP servers, skills, agents e rules.

name, version, authorString
manifestJSON {}
sourcemanual | imported
Has many: mcpServers, skills, agents, rules

Webhook

Webhook outbound com secret HMAC-SHA256 e retry.

urlString
eventsJSON[] event names
secretString (HMAC key)
Has many: deliveries (WebhookDelivery)

WebhookDelivery

Registro de entrega com retry (max 3, backoff exponencial).

payloadString JSON
responseCode, responseBodyResult
attemptsInt (max 3)
statuspending|success|failed

Sistema — Hooks, API Keys, Git

Hook

Claude Code Hook: PreToolUse, PostToolUse, Stop. Handler: command/prompt/agent.

eventTypePreToolUse | PostToolUse | Stop
matcherString? regex
handlerTypecommand | prompt | agent
command, promptString?
timeoutInt (60000ms)
isGlobal, isTemplateBoolean

ApiKey

API Key (exk_* prefix). Hash SHA-256 no DB.

nameString
keyHashString @unique SHA-256
prefixString (exk_abc1)
expiresAtDateTime?
revokedBoolean

GitAccount

Conta GitHub multi-account (Pessoal, Trabalho). Token em plain text.

labelString (Pessoal, Trabalho)
tokenString ⚠️ plain text
usernameString? cached
Has many: projectMappings

ProjectGitMapping

Mapeia projectPath → GitAccount específica.

projectPathString @unique
gitAccountIdFK → GitAccount

Redis — Cache, Pub/Sub & Filas

Cache geral, BullMQ queue, estado temporário. Lazy init.

Subscriber Connection

Subscribe em channels execution:{conversationId}. Eventos em tempo real.

Publisher Connection

Publish de eventos do orchestrator para SSE consumers.

BullMQ — Fila "execution"

Concurrency: 3 removeOnComplete: 100 removeOnFail: 50 Retry: disabled

Notas de Segurança

✓

Senhas: bcryptjs (10 rounds) — campo passwordHash

✓

API Keys: SHA-256 hash — campo keyHash (original nunca armazenado)

✗

GitHub Tokens: plain text — campos User.githubToken e GitAccount.token

✓

Cascade Delete: Todos os recursos deletam em cascata quando o User é removido

✓

Índices: 30+ índices otimizando queries por userId, conversationId, executionId, stepId
