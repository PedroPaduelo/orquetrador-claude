O que sao Agents

Um Agent e um executor inteligente com seu proprio system prompt, conjunto de tools permitidas/bloqueadas, modo de permissao e limite de turnos. Diferente de uma Skill (que e um bloco de instrucao reutilizavel), o Agent define um contexto de execucao completo.

Agents podem ter Skills acopladas (relacao M:M via AgentSkill), herdar configuracoes de um Plugin, e ser anexados a workflow steps via WorkflowStepAgent. No momento da execucao, o system prompt do Agent e mesclado com os prompts das skills e o contexto do step.

Relacoes

🔗
AgentSkill (M:M)

Tabela join agent_skills. Campos: agentId, skillId, order, config (Json). Unique constraint [agentId, skillId]. Cascade em ambos os lados.

⚙
WorkflowStepAgent

Tabela join workflow_step_agents. Liga Agent a um WorkflowStep. Campos: stepId, agentId, order, config.

🧩
Plugin (optional)

Se importado de um Plugin, pluginId aponta para Plugin.id. onDelete: SetNull — agent sobrevive se plugin for removido.

API Endpoints (8)

GET /agents
Lista todos os agents do usuario. Retorna array com id, name, description, model, permissionMode, maxTurns, tools, enabled, isGlobal, pluginId, source, repoUrl, lastSyncedAt.
POST /agents
Cria novo agent. Body: name (obrigatorio), description, systemPrompt, tools[], disallowedTools[], model, permissionMode, maxTurns, skills[] (IDs), enabled, isGlobal. Retorna 201.
GET /agents/:id
Busca agent por ID com detalhes completos: systemPrompt, tools, disallowedTools, skills associadas, plugin, timestamps.
PUT /agents/:id
Atualiza agent. Todos os campos opcionais. Verifica ownership antes de alterar. Retorna id, name, updatedAt.
DELETE /agents/:id
Remove agent. Verifica ownership. Retorna 204. Cascade remove AgentSkill e WorkflowStepAgent associados.
PATCH /agents/:id/toggle
Alterna enabled/disabled. Retorna id e novo estado enabled.
POST /agents/import
Importa agent de URL (raw markdown) ou conteudo colado. Body: url?, content?, isGlobal. Faz parse do markdown, extrai system prompt e metadata. Retorna 201.
POST /agents/:id/resync
Re-sincroniza agent importado com a fonte GitHub. Atualiza systemPrompt, tools e metadata. Retorna id, name, lastSyncedAt.

MCP Tools (7)

agent_list Lista todos os agents configurados. Retorna array com id, name, description, model, enabled, isGlobal, permissionMode.
agent_get Busca agent por ID com detalhes completos: systemPrompt, tools, disallowedTools, skills associadas, model, permissionMode, maxTurns.
agent_create Cria agent com name, systemPrompt, tools, disallowedTools, model, permissionMode, maxTurns, skills (IDs), enabled, isGlobal.
agent_update Atualiza campos do agent. Todos opcionais. Recomendacao: use agent_get antes para ver estado atual.
agent_delete Remove agent por ID. Cascade nas relacoes AgentSkill e WorkflowStepAgent.
agent_toggle Alterna estado enabled/disabled de um agent.
agent_import Importa agent de URL (markdown raw) ou conteudo colado. Parse automatico de system prompt e metadata.

Uso em Workflows

Agents sao anexados a WorkflowSteps via a tabela join WorkflowStepAgent. Cada step pode ter multiplos agents (ordenados por order), e cada agent pode ser reutilizado em multiplos steps.

Durante a execucao de um step, o sistema:

Fluxo de Execucao do Agent em um Workflow Step
1. Carrega WorkflowStep
↓
2. Busca agents via WorkflowStepAgent (order ASC)
↓
3. Merge: systemPrompt + skills do agent
↓
4. Aplica tools[] e disallowedTools[]
↓
5. Define permissionMode e maxTurns
↓
6. Envia para Claude CLI com contexto completo
// WorkflowStepAgent — tabela join model WorkflowStepAgent { id String @id @default(uuid()) stepId String @map("step_id") agentId String @map("agent_id") order Int @default(0) config Json? step WorkflowStep @relation(...) agent Agent @relation(...) @@unique([stepId, agentId]) @@index([agentId]) @@map("workflow_step_agents") }

📂
GitHub Integration

📥
Import

POST /agents/import aceita URL de raw markdown ou conteudo direto. O service faz parse do markdown, extrai metadata (nome, descricao, model, tools) e cria o agent com source: "imported".

Campos GitHub preenchidos automaticamente: repoOwner, repoName, repoBranch, repoPath.

🔄
Re-sync

POST /agents/:id/resync re-busca o conteudo da fonte GitHub original. Atualiza systemPrompt, tools, description e seta lastSyncedAt.

Mesmo padrao usado em Skills — tracking estruturado com campos repoOwner/repoName/repoBranch/repoPath.

🚀
Fluxo de Uso

  1. Acesse a pagina Agents no frontend
  2. Clique em "Novo Agent" → abre AgentModal
  3. Preencha: nome, descricao, system prompt
  4. Configure tools permitidas e bloqueadas
  5. Selecione model, permissionMode e maxTurns
  6. Associe Skills existentes (multi-select)
  7. Salve → POST /agents → agent criado com source: "manual"
  1. Clique em "Importar" → abre ImportAgentDialog
  2. Cole a URL do raw markdown do GitHub (ou cole o conteudo)
  3. Sistema faz parse: extrai nome, descricao, system prompt, tools
  4. Preview mostra o que sera importado → confirme
  5. POST /agents/import → agent criado com source: "imported"
  6. Campos GitHub preenchidos automaticamente para re-sync futuro
  1. Edite um Workflow → selecione um Step
  2. Na aba "Agents" do step, adicione um ou mais agents
  3. Defina a ordem de execucao (campo order)
  4. Ao executar o workflow, o step carrega seus agents na ordem
  5. System prompt do agent e mesclado com skills e contexto do step
  6. Tools e permissionMode do agent sao aplicados na execucao Claude
