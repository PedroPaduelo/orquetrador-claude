Visao Geral

O Orquetrador Claude oferece integracao completa com o GitHub, permitindo que usuarios importem e sincronizem skills, agents, rules e hooks diretamente de repositorios Git. Cada usuario pode configurar multiplas contas GitHub (pessoal, trabalho), e o sistema rastreia a origem de cada recurso importado para manter sincronizacao bidirecional.

📥
Import Flow

Fluxo de Importacao de Repositorio

1
Usuario fornece URL do repositorio
Via ImportRepoDialog ou import dialog por tipo de recurso
2
Sistema escaneia o repositorio
Busca arquivos de skills (.md, .txt), agents (CLAUDE.md), rules e hooks via GitHub API
3
Cria resources com source='imported'
Skills, Agents, Rules sao criados no banco com flag de origem
4
Registra metadata no GitSource
repoOwner, repoName, repoBranch, repoPath, lastSyncedAt
5
Cria fileManifest[] para cada recurso
Lista de arquivos com paths e hashes para deteccao de mudancas

🔄
Sync Flow

Fluxo de Sincronizacao

1
PATCH /resources/:type/:id/sync
Usuario ou scheduler dispara sincronizacao
2
Busca GitSource do recurso
Localiza metadata: repo, branch, path
3
Fetch latest do GitHub
Baixa conteudo atualizado via GitHub REST API
4
Compara com fileManifest
Detecta diferencas entre versao local e remota
5
Atualiza resource + lastSyncedAt
body/content atualizado, manifest renovado, timestamp atualizado

Seguranca

🔐
Token Encryption

Tokens GitHub sao criptografados com AES-256-CBC antes de salvar no banco. A chave de criptografia vem da env ENCRYPTION_KEY.

// git.routes.ts const encryptedToken = encrypt(token); await prisma.gitAccount.create({ data: { token: encryptedToken, ... } }); // Na leitura const token = decrypt(account.token);
🛡
Ownership & Isolation

Todas as queries Git filtram por userId. Um usuario so ve e sincroniza recursos vinculados as suas contas. Operacoes de update/delete verificam ownership em transacao antes de executar.

userId filter $transaction ownership AES encrypt/decrypt

File Manifest

O fileManifest e um array JSON armazenado em cada recurso importado. Ele registra todos os arquivos que compoem o recurso, permitindo deteccao precisa de mudancas durante o sync.

// Exemplo de fileManifest para uma Skill importada [ { "path": "skills/code-review.md", "sha": "a1b2c3d4e5f6...", "size": 2048, "type": "file" }, { "path": "skills/code-review-examples/", "sha": "f6e5d4c3b2a1...", "size": 0, "type": "dir" } ] // Comparacao durante sync: // 1. Fetch current SHA de cada arquivo via GitHub API // 2. Compara com SHA armazenado no manifest // 3. Se diferente → atualiza conteudo + SHA // 4. Se igual → skip (nenhuma mudanca)
