Critico 4 vulnerabilidades

CORS Aberto

CVE-like

origin: true — qualquer dominio acessa a API

// backend/src/app.ts
app.register(cors, { origin: true })
Permite CSRF cross-origin e data exfiltration

Rate Limiting Desabilitado

DoS Risk

Rate limiting registrado mas desabilitado globalmente

// @fastify/rate-limit
global: false // nao aplica a nenhuma rota
Brute force, credential stuffing, resource exhaustion

Sem Token Refresh

Auth Gap

Tokens JWT de 7 dias sem nenhum mecanismo de rotacao

// JWT sign options
{ expiresIn: '7d' } // no refresh, no revocation
Token roubado = acesso por 7 dias sem possibilidade de revogacao

GitHub Tokens em Plain Text

Data Leak

Tokens do GitHub armazenados sem criptografia no banco de dados

// prisma schema
githubToken String? // plain text in DB!
DB breach = acesso total aos repos GitHub dos usuarios
Alto 3 vulnerabilidades

Sem Security Headers

Nenhum header de seguranca configurado:

  • x HSTS
  • x Content-Security-Policy
  • x X-Frame-Options
  • x X-Content-Type-Options

localStorage para JWT

Tokens JWT armazenados em localStorage

localStorage.setItem('token', jwt)

Vulneravel a XSS — qualquer script injetado rouba o token

JWT_SECRET Default Fraco

Fallback para secret hardcoded quando env nao esta setada

JWT_SECRET = process.env.JWT_SECRET
  || 'default-secret'

Permite forjar tokens se env nao estiver configurada

Medio 3 vulnerabilidades

Sem HTTPOnly Cookies

Tokens nunca sao armazenados em cookies HTTPOnly. Toda autenticacao via headers manuais.

Sem Protecao CSRF

Nenhum token CSRF. Combinado com CORS aberto, permite requisicoes forjadas cross-origin.

API Keys Exibidas Uma Vez

A key completa e exibida apenas na criacao. Apos isso, apenas o hash fica no banco. Se o usuario perder, precisa gerar outra.
