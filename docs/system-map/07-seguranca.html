⚠ 6 Riscos Mapeados

Todos (6) ● Alto (1) ● Medio (3) ● Baixo (2)
● Alto Risk #1
Exposicao de Tokens/Secrets em Traces
stdout e stderr de execucoes podem conter tokens, API keys e secrets. Os dados de trace sao armazenados no banco e ficam acessiveis via API de execucoes.
Impacto
Vazamento de credenciais em logs de execucao pode comprometer sistemas externos integrados.
Mitigacao sugerida
Implementar sanitizacao de output com regex para padroes de secrets (tokens, passwords, API keys) antes de persistir traces.
● Medio Risk #2
Sandbox Escape
O Claude CLI tem acesso ao sistema como usuario sandbox. Um prompt injection ou uso criativo do CLI pode escapar do isolamento previsto.
Impacto
Execucao de comandos arbitrarios no container com permissoes do sandbox user.
Mitigacao sugerida
Aplicar cgroups, namespaces ou containers adicionais para limitar recursos e acesso a filesystem.
● Medio Risk #3
API Key Leakage
API keys podem estar expostas em variaveis de ambiente, memoria do processo ou logs de debug. Embora o hash SHA256 proteja o armazenamento, a chave original transita em runtime.
Impacto
Acesso nao autorizado a API se a chave for capturada em transito ou via memory dump.
Mitigacao sugerida
Implementar rotacao automatica de keys, reduzir TTL, usar HMAC em vez de comparacao direta.
● Baixo Risk #4
CORS Misconfiguration
Configuracao atual usa CORS_ORIGINS com fallback para dominios do projeto. Anteriormente era origin: true (permissivo). Ja corrigido, mas requer monitoramento.
Impacto
CSRF ou acesso cross-origin indesejado se a lista de origens permitidas ficar desatualizada.
Status
Corrigido — CORS_ORIGINS env var com lista explicita de dominios permitidos.
● Medio Risk #5
Rate Limiting Desabilitado
O rate limit global (@fastify/rate-limit) esta configurado mas desabilitado por padrao. APIs ficam vulneraveis a brute-force e DDoS.
Impacto
Ataques de forca bruta em login, abuse de API keys, consumo excessivo de recursos.
Mitigacao sugerida
Habilitar rate limit global com config adequada (ex: 100 req/min geral, 10 req/min em /auth/login).
● Baixo Risk #6
Webhook Secret Management
Webhook secrets eram armazenados como plain text no banco. Agora utilizam encrypt() via ENCRYPTION_KEY, mas a feature e opcional (fallback sem criptografia).
Impacto
Secrets visiveis em banco se ENCRYPTION_KEY nao estiver configurada.
Status
Parcialmente corrigido — encrypt/decrypt implementados, mas ENCRYPTION_KEY deve ser obrigatoria.

🚨 Error Handling Security

🚫
No Stack Traces
Stack traces nunca expostos em producao. Erros retornam apenas mensagem generica + code.
📌
Custom Error Classes
AppError, ValidationError, NotFoundError, UnauthorizedError com status codes padronizados.
🗃
Prisma Error Mapping
Erros Prisma (P2002, P2025, etc.) mapeados para HTTP status codes sem expor detalhes internos.
Error Class HTTP Status Quando usar
ValidationError 400 Input invalido, campos faltando, formato incorreto
UnauthorizedError 401 Token invalido/expirado, API key revogada
ForbiddenError 403 Role sem permissao para a acao
NotFoundError 404 Recurso nao existe ou nao pertence ao usuario
ConflictError 409 Prisma P2002 — unique constraint violation
AppError 500 Erro interno generico (sem detalhes ao cliente)
// Error handler global app.setErrorHandler(async (error, request, reply) => { if (error instanceof AppError) { return reply.status(error.statusCode).send({ error: error.code, message: error.message }) } // Prisma errors mapping if (error.code === 'P2002') { return reply.status(409).send({ error: 'CONFLICT' }) } // Generic 500 — NO stack trace reply.status(500).send({ error: 'INTERNAL_ERROR' }) })

🔐 Encryption

Dados criptografados (AES)
✓ Git tokens (via encrypt() em git.routes)
✓ Webhook secrets (via encrypt() em webhooks.repository)
✓ Trigger webhook secrets (via encrypt() em triggers.repository)
✓ Decriptacao transparente ao ler (decrypt())
Configuracao
⚠ ENCRYPTION_KEY env var — opcional (deveria ser obrigatoria)
⚠ GitHub tokens armazenados como TEXT no schema (campo String no Prisma)
✓ Fallback sem criptografia se ENCRYPTION_KEY ausente
import { createCipheriv, createDecipheriv, randomBytes } from 'crypto' export function encrypt(text: string): string { const key = process.env.ENCRYPTION_KEY if (!key) return text // fallback: sem criptografia const iv = randomBytes(16) const cipher = createCipheriv('aes-256-cbc', Buffer.from(key, 'hex'), iv) const encrypted = Buffer.concat([cipher.update(text), cipher.final()]) return iv.toString('hex') + ':' + encrypted.toString('hex') }

📑 Audit Trail

Toda acao relevante no sistema e registrada na tabela AuditLog, permitindo rastreabilidade completa de operacoes administrativas e de seguranca.
Campo Tipo Descricao
id UUID Identificador unico do log
action String Acao realizada (ex: user.create, workflow.delete)
resourceType String Tipo do recurso (User, Workflow, Execution, ApiKey, etc.)
resourceId UUID ID do recurso afetado
diff JSON Diff das alteracoes (before/after)
ipAddress String IP de origem da requisicao
userId UUID Usuario que executou a acao
createdAt DateTime Timestamp da acao
// Registrar audit log await prisma.auditLog.create({ data: { action: 'apikey.revoke', resourceType: 'ApiKey', resourceId: apiKeyId, diff: { status: { before: 'active', after: 'revoked' } }, ipAddress: request.ip, userId: request.user.id, } })
