? O que e DAG

DAG significa Directed Acyclic Graph (Grafo Dirigido Aciclico). No contexto do Orquetrador, e o mecanismo que permite executar steps de um workflow em paralelo sempre que suas dependencias ja estiverem satisfeitas.

D
Directed
Cada aresta tem direcao — step A deve rodar antes de step C
A
Acyclic
Sem ciclos — nenhum step pode depender de si mesmo (direta ou indiretamente)
G
Graph
Estrutura de nos (steps) conectados por arestas (dependencias)

Steps sem dependencias rodam simultaneamente. Um step so inicia quando todos os steps de que ele depende estiverem completos. Isso reduz drasticamente o tempo total de execucao de workflows complexos.

C Classe DAGExecutor

F
dag-executor.ts
105 linhas | backend/src/domains/execution/orchestrator/

Classe responsavel por modelar o grafo de dependencias entre steps, validar a estrutura (sem ciclos ou refs invalidas), determinar quais steps estao prontos para execucao e rastrear outputs para injecao de contexto.

Estado Interno

private graph: Map<string, DagStep> // mapa stepId -> {step, index, dependsOn} private completed = new Set<string>() // IDs dos steps finalizados private outputs = new Map<string, string>() // stepId -> output (texto)

Metodos

constructor(steps)
steps: WorkflowStep[]
Recebe o array de steps do workflow e constroi o grafo de dependencias. Para cada step, le o campo dependsOn (JSON array de IDs) e cria uma entrada no Map interno.
validate()
-> { valid: boolean; error?: string }
Verifica integridade do grafo: (1) todas as refs de dependencia apontam para steps existentes, (2) nao existem ciclos (DFS com deteccao de back-edge). Retorna erro descritivo se falhar.
getReadySteps()
-> DagStep[]
Retorna steps prontos para execucao: aqueles que ainda nao foram completados E cujas dependencias ja estao todas no set completed. Estes formam o proximo batch.
markCompleted(stepId, output)
stepId: string, output: string
Marca um step como finalizado e armazena seu output. Isso pode desbloquear novos steps cujas dependencias agora estao satisfeitas.
getDependencyContext(stepId)
-> string
Retorna concatenacao dos outputs de todos os steps de que stepId depende. Formatado como [Output from "Nome"]: texto. Injetado no contexto da execucao do step.
isComplete()
-> boolean
Retorna true quando completed.size === graph.size, ou seja, todos os steps do grafo foram finalizados com sucesso.

D Deteccao de DAG

Funcao exportada isDAGWorkflow(steps)

Determina se um workflow deve usar execucao DAG ou sequencial. Retorna true se qualquer step do array possuir dependsOn com pelo menos um elemento.

export function isDAGWorkflow(steps: WorkflowStep[]): boolean { return steps.some(step => { const deps = step.dependsOn as string[] || [] return Array.isArray(deps) && deps.length > 0 }) }
Retorna true
Pelo menos um step tem dependsOn: ["id-1"]
Retorna false
Todos os steps tem dependsOn: [] ou ausente

F Fluxo de Execucao

1. Detectar modo DAG
isDAGWorkflow(steps) verifica se algum step possui dependsOn. Se false, usa execucao sequencial normal.
2. Construir grafo
new DAGExecutor(steps) monta o Map interno com todos os steps e suas dependencias.
3. Validar DAG
validate() checa refs invalidas e ciclos. Se falhar, a execucao e abortada com erro descritivo.
4. Loop de execucao
Enquanto !isComplete():
a) getReadySteps() retorna steps com deps satisfeitas
b) Emite evento SSE dag_batch_start
c) Executa batch (steps rodam em paralelo ou sequencial, conforme config)
d) markCompleted(stepId, output) para cada step finalizado
5. Injecao de contexto
Antes de executar cada step, getDependencyContext(stepId) coleta outputs dos steps predecessores e injeta como contexto adicional na mensagem enviada ao Claude.
6. Conclusao
isComplete() retorna true quando todos os steps estao no set completed. Workflow finalizado.

C Contexto de Dependencias

O metodo getDependencyContext(stepId) permite que cada step tenha acesso aos resultados dos steps de que depende. O output e formatado e injetado como mensagem adicional no contexto da execucao.

V Regras de Validacao

1
Dependencias Ausentes

Todos os IDs referenciados em dependsOn devem existir como steps reais no workflow. Se um step referencia um ID inexistente, a validacao falha.

// Erro retornado: { valid: false, error: 'Step "Deploy" depends on unknown step ID "abc-123"' }
2
Deteccao de Ciclos

Ciclos causariam loop infinito. A validacao usa DFS (Depth-First Search) com deteccao de back-edge via set inStack. Se um no ja no stack de recursao e revisitado, ha um ciclo.

// Ciclo: A -> B -> C -> A { valid: false, error: 'Cycle detected in step dependencies' }

Algoritmo de Deteccao de Ciclos (DFS)

const visited = new Set<string>() // nos ja totalmente processados const inStack = new Set<string>() // nos na pilha de recursao atual const hasCycle = (id: string): boolean => { if (inStack.has(id)) return true // back-edge = ciclo! if (visited.has(id)) return false // ja processado, sem ciclo visited.add(id) inStack.add(id) for (const dep of node.dependsOn) { if (hasCycle(dep)) return true } inStack.delete(id) // remove da pilha ao voltar return false }

E Eventos SSE

1
dag_batch_start
Emitido a cada batch

Emitido quando um novo batch de steps inicia execucao. Permite ao frontend mostrar progresso do DAG em tempo real.

{ "type": "dag_batch_start", "data": { "steps": ["Step A", "Step B", "Step D"], "batchNumber": 1 } }
2
step_blocked
Emitido quando step aguarda

Emitido quando um step nao pode ser executado porque uma ou mais dependencias ainda nao foram completadas.

{ "type": "step_blocked", "data": { "stepName": "Step E", "blockedBy": ["Step C", "Step D"] } }

D WorkflowStep.dependsOn

Campo JSON no model WorkflowStep do Prisma. Armazena um array de IDs (UUIDs) de outros steps do mesmo workflow.

Valor Significado Comportamento
[] ou ausente Sem dependencias Step roda no primeiro batch disponivel
["uuid-1"] Depende de 1 step Roda apos uuid-1 completar
["uuid-1", "uuid-2"] Depende de 2 steps Roda apos AMBOS completarem

Exemplo no Banco

-- Step sem dependencias INSERT INTO workflow_step (id, name, dependsOn, ...) VALUES ('step-a', 'Analisar Codigo', '[]', ...); -- Step com dependencias INSERT INTO workflow_step (id, name, dependsOn, ...) VALUES ('step-c', 'Gerar Relatorio', '["step-a", "step-b"]', ...);

I Interface DagStep

Tipo interno usado pelo DAGExecutor para representar cada no do grafo.

interface DagStep { step: WorkflowStep // objeto completo do Prisma index: number // posicao original no array dependsOn: string[] // array de step IDs (UUIDs) }
