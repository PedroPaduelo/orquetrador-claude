🏛 Padroes Arquiteturais

O sistema adota 9 padroes arquiteturais que garantem escalabilidade, resiliencia e manutenibilidade.

🗃
Repository Pattern
Camada de abstracao entre servicos e banco de dados. Cada dominio tem seu repository.ts que encapsula queries Prisma, facilitando testes e troca de ORM.
Data Access
📢
Event-Driven (EventEmitter)
Componentes emitem eventos desacoplados via Node.js EventEmitter. Hooks, triggers e monitoramento escutam esses eventos sem acoplamento direto.
Messaging
🚀
Fire-and-Forget
Operacoes nao-criticas (logging, metricas, notificacoes) sao disparadas sem aguardar resposta, mantendo a latencia do request principal baixa.
Performance
🛡
Graceful Degradation
Redis e opcional. Se indisponivel, o sistema continua operando sem cache, filas ou pub/sub. Funcionalidades degradam sem derrubar o servico.
Resiliencia
🔄
State Machines
Workflows, execucoes e conversations seguem maquinas de estado com transicoes validas definidas. Garante consistencia e previne estados invalidos.
Consistencia
📦
Async Job Queue (BullMQ)
Tarefas pesadas (execucoes Claude, imports) sao enfileiradas no BullMQ via Redis. Workers processam em background com retry, backoff e dead-letter.
Async Processing
📡
SSE Streaming
Server-Sent Events para streaming de output em tempo real. O frontend recebe tokens incrementais durante execucoes Claude sem polling.
Real-time
🚨
Error Categories
Erros sao categorizados (validation, auth, not-found, conflict, internal) com codigos e mensagens padronizados. Respostas HTTP consistentes em toda API.
Error Handling
💰
Token Budgeting
Controle de orcamento de tokens por workflow, execucao e usuario. Limites configurados previnem gastos excessivos com chamadas ao Claude.
Cost Control
