🏷 Universal Tagging System

Tags podem ser aplicadas a qualquer tipo de recurso do sistema. O modelo polimórfico ResourceTag conecta tags a recursos de forma genérica, sem necessidade de tabelas join específicas para cada tipo.

Arquitetura de Tagging
🏷 Tag
→
🔗 ResourceTag
→
📦 Any Resource

📋 Tag Model

🏷
Tag
Modelo principal
model Tag { // Campos id String @id @default(uuid()) name String // Unico por usuario color String // Hex (#FF5733) userId String createdAt DateTime @default(now()) updatedAt DateTime @updatedAt // Relations user User @relation resources ResourceTag[] // Polimorfismo @@unique([name, userId]) }
🔗
ResourceTag
Join polimórfico
model ResourceTag { // Campos id String @id @default(uuid()) tagId String resourceType ResourceType // Enum resourceId String createdAt DateTime @default(now()) // Relations tag Tag @relation @@unique([tagId, resourceType, resourceId]) }
ResourceType Enum
enum ResourceType { mcp_server skill agent rule hook plugin workflow }

🛡 Resource Dependencies

O modelo ResourceDependency rastreia dependências entre recursos. Usado para validar se é seguro deletar um recurso (verifica se outros dependem dele).

🔗
ResourceDependency
Grafo de dependências
model ResourceDependency { id String @id @default(uuid()) sourceType ResourceType // Quem depende sourceId String dependencyType ResourceType // De quem depende dependencyId String dependencyName String // Nome legivel isOptional Boolean @default(false) createdAt DateTime @default(now()) }
Exemplo: Grafo de Dependências
🔄 Workflow “Deploy”
sourceType: workflow
→ depends on
🤖 Agent “Code Reviewer”
dependencyType: agent
🔄 Workflow “Deploy”
sourceType: workflow
→ depends on
🧩 Plugin “Slack Notifier”
dependencyType: plugin optional
🧠 Skill “Analyze PR”
sourceType: skill
→ depends on
🖥 MCP Server “GitHub”
dependencyType: mcp_server
Dependency Endpoints
2 endpoints
GET /resources/:type/:id/dependents Quem depende deste recurso
GET /resources/:type/:id/validate-deletion Seguro deletar?
GET /resources/:type/:id/validate-deletion
Safe to delete
{ "safe": true, "dependents": [], "message": "Nenhum recurso depende deste" }
NOT safe to delete
{ "safe": false, "dependents": [ { "sourceType": "workflow", "sourceId": "wf-001", "isOptional": false } ], "message": "1 recurso depende" }

📄 Step Templates

Templates reutilizáveis para configuração de steps em workflows. Permitem padronizar steps comuns sem recriá-los manualmente.

🔄 Resource Lifecycle

1
Create
Recurso criado via API. Tipo registrado no enum ResourceType.
2
Tag
Tags associadas via POST /tags/:id/resources. Permite organizar e filtrar.
3
Use
Recurso utilizado em workflows. Dependencias registradas automaticamente.
4
Track
ResourceDependency rastreia quem depende de quem.
5
Delete
validate-deletion verifica dependentes antes de permitir a remocao.
