ⓘ Visao Geral

🔌
Model Context Protocol
Padrao aberto da Anthropic para integracao de ferramentas com modelos de IA. Funciona como um "USB" universal — qualquer ferramenta que implemente MCP pode ser consumida por qualquer cliente compativel (Claude CLI, Claude Desktop, IDEs, scripts).
  • Protocolo padronizado de tool calling
  • JSON-RPC 2.0 sobre HTTP
  • Sessoes stateful com gerenciamento de estado
⚙
Este Servidor
Expoe toda a funcionalidade do Orquetrador como ferramentas MCP. Qualquer operacao possivel na UI tambem e possivel via MCP.
Transport StreamableHTTPServerTransport na porta 3334
Auth Bearer token (API key) por sessao
Backend Proxy para API REST Fastify (:3333)
SDK @modelcontextprotocol/sdk v1.27.0

🔒 Protocolo MCP

📡
Endpoints HTTP
MetodoRotaDescricaoHeaders
POST /mcp Inicializacao de sessao e chamadas de tools (JSON-RPC) Authorization: Bearer <api-key>
GET /mcp Stream SSE — recebe eventos em tempo real da sessao mcp-session-id: <uuid>
DELETE /mcp Encerra sessao e limpa recursos mcp-session-id: <uuid>
GET /health Health check do servidor MCP —
🗝
Session Management
Cada conexao recebe um UUID unico de sessao. O servidor mantem um Map<string, ServerSession> em memoria.
// Fluxo de sessao POST /mcp { method: "initialize" } → Servidor gera UUID → Cria ServerSession → Retorna mcp-session-id header POST /mcp { method: "tools/call", ... } → Header: mcp-session-id: <uuid> → Lookup sessao no Map → Executa tool com API key da sessao DELETE /mcp → Remove sessao do Map → Limpa backgroundExecutions
🔐
Per-Session Authentication
A API key e extraida do header Authorization: Bearer na requisicao de inicializacao e vinculada a sessao.
// Autenticacao por sessao const apiKey = req.headers.authorization ?.replace('Bearer ', ''); // Armazenada na sessao session.apiKey = apiKey; // Usada em TODA chamada ao backend apiGet('/api/workflows', session.apiKey); apiPost('/api/agents', body, session.apiKey);

🚀 API Client Layer

📦
Helpers HTTP
Funcoes auxiliares que encapsulam fetch para o backend:
function apiGet(path, apiKey) { ... } function apiPost(path, body, apiKey) { ... } function apiPut(path, body, apiKey) { ... } function apiPatch(path, body, apiKey) { ... } function apiDelete(path, apiKey) { ... }

Todos adicionam Authorization: Bearer automaticamente e fazem JSON parse da resposta.

📡
apiPostSSE — Streaming
Helper especial para Server-Sent Events com timeout de 600 segundos:
async function apiPostSSE( path: string, body: object, apiKey: string, onChunk: (data) => void ): Promise<string> // Timeout: 600s (10 min) // Acumula chunks SSE // Retorna resposta completa // Usado por send_message (sync)

⚡ Background Execution Pattern

O padrao async/await permite execucoes paralelas — essencial para swarms de agentes onde multiplas conversas rodam simultaneamente.

1
conversation_send_message_async Inicia SSE em background thread
2
Promise armazenada em backgroundExecutions Map<convId, Promise>
3
Retorna imediatamente — { status: "started", conversationId }
4
conversation_await_execution Aguarda promise do Map e retorna resultado completo
// Execucao paralela de 3 agentes await conversation_send_message_async(convA, "task 1") await conversation_send_message_async(convB, "task 2") await conversation_send_message_async(convC, "task 3") // Aguardar resultados (em qualquer ordem) resultA = await conversation_await_execution(convA) resultB = await conversation_await_execution(convB) resultC = await conversation_await_execution(convC)
