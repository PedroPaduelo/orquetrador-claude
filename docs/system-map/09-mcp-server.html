O que e MCP?

Model Context Protocol

Padrao aberto da Anthropic para integracao de ferramentas com modelos de IA. Funciona como um "USB" universal — qualquer ferramenta que implemente MCP pode ser consumida por qualquer cliente compativel (Claude CLI, Claude Desktop, IDEs, scripts).

Protocolo padronizado de tool calling
JSON-RPC 2.0 sobre HTTP
Sessoes stateful com gerenciamento de estado

Expoe TODA a funcionalidade do Orquetrador como ferramentas MCP. Qualquer operacao possivel na UI tambem e possivel via MCP — criar workflows, executar conversas, gerenciar agentes, orquestrar swarms.

Ferramentas por Categoria

Orchestration Pattern — Swarm de Agentes

As ferramentas MCP permitem que um agente orquestrador crie e encadeie multiplas conversas, formando um swarm de agentes especializados. Cada conversa usa um workflow diferente e o output de uma alimenta a proxima.

Encadeamento Sequencial

1
conversation_create(workflow_A) → ID_A
2
conversation_send_message(ID_A, "task 1") → result_A SYNC - aguarda
3
conversation_create(workflow_B) → ID_B
4
conversation_send_message(ID_B, result_A) → result_B encadeia output
5
conversation_create(workflow_C) → ID_C
6
conversation_send_message(ID_C, result_A + result_B) → review consolida

Execucao Paralela (Async)

1
conversation_send_message_async(ID_A, task) retorna imediato
2
conversation_send_message_async(ID_B, task) retorna imediato
3
conversation_await_execution(ID_A) → result_A
4
conversation_await_execution(ID_B) → result_B
5
Usa result_A + result_B na proxima etapa

Diagrama de Conexao

Gerenciamento de Sessao

1. Initialize
POST /mcp
Headers:
Authorization: Bearer <api-key>
Body: { initialize request }
Response header:
mcp-session-id: <uuid>

Cria uma nova sessao MCP. O servidor gera um sessionId unico e armazena a API key associada.

2. Tool Call
POST /mcp
Headers:
mcp-session-id: <uuid>
Authorization: Bearer <api-key>
Body: { tools/call request }

Cada chamada de tool envia o mcp-session-id no header. O servidor roteia para a sessao correta e usa a API key armazenada.

3. Cleanup
DELETE /mcp
Headers:
mcp-session-id: <uuid>
Response:
{ "success": true }

Fecha a sessao e remove a API key armazenada. O transport e destruido e os recursos liberados.

Como Conectar 

{
  "mcpServers": {
    "orquetrador": {
      "type": "http",
      "url": "https://orquestrador-mcp.ddw1sl.easypanel.host/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY"
      }
    }
  }
}
// 1. Initialize session
const initRes = await fetch("https://orquestrador-mcp.ddw1sl.easypanel.host/mcp", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY",
  },
  body: JSON.stringify({
    jsonrpc: "2.0",
    id: 1,
    method: "initialize",
    params: {
      protocolVersion: "2025-03-26",
      capabilities: {},
      clientInfo: { name: "my-client", version: "1.0" }
    }
  })
});

const sessionId = initRes.headers.get("mcp-session-id");

// 2. Call a tool
const toolRes = await fetch("https://orquestrador-mcp.ddw1sl.easypanel.host/mcp", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "mcp-session-id": sessionId,
  },
  body: JSON.stringify({
    jsonrpc: "2.0",
    id: 2,
    method: "tools/call",
    params: {
      name: "health_check",
      arguments: {}
    }
  })
});
import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StreamableHTTPClientTransport } from "@modelcontextprotocol/sdk/client/streamableHttp.js";

const client = new Client({
  name: "my-orchestrator-client",
  version: "1.0.0",
});

const transport = new StreamableHTTPClientTransport(
  new URL("https://orquestrador-mcp.ddw1sl.easypanel.host/mcp"),
  {
    requestInit: {
      headers: { "Authorization": "Bearer YOUR_API_KEY" }
    }
  }
);

await client.connect(transport);

// List tools
const { tools } = await client.listTools();
console.log(`\${tools.length} tools available`);

// Call a tool
const result = await client.callTool({
  name: "workflow_list",
  arguments: {}
});
console.log(result);
