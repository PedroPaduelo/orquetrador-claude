sudo -n -u claude-sandbox -g dev --preserve-env -- claude [args] // Env blocklist (sensitive vars) DATABASE_URL, JWT_SECRET GROQ_API_KEY, CLAUDECODE CLAUDE_CODE_ENTRYPOINT npm_lifecycle_*, npm_package_*

JSON-stream events for parsing

Session Continuity & Output

// Resume token extracted from stream-json event: type: "session" sessionId: "session_..." // StreamParser handles: type: "content" // text chunks type: "action" // tool_use, tool_result, etc. type: "error" // error events type: "session" // resume token

Isolamento: Cada execucao roda como claude-sandbox via sudo. Arquivos e diretorios do projeto sao configurados com mode 0o775 para permitir escrita pelo sandbox. Variaveis sensiveis sao removidas do ambiente via blocklist.

@anthropic-ai/sdk via loadbalance-back.ddw1sl.easypanel.host

1. LLM Judge Validator

Model claude-haiku-4-5-20251001
Max Tokens 200
Output PASS / FAIL

Avalia saida de steps contra criterios customizados. Retorna valid: true/false com detalhes. Se falhar, nao bloqueia a execucao (fail-open).

2. Memory Manager

Model claude-haiku-4-5-20251001
Max Tokens 4000
Storage StepMemory (Prisma)

Resume contexto acumulado de conversas longas. Extrai: pedidos, implementacoes, estado, arquivos, decisoes. Persiste em step_memories table.

Outbound

Webhook System

HMAC-SHA256, exponential backoff, delivery tracking

Security & Payload

X-Webhook-Signature: sha256={hmac} X-Webhook-Event: execution Content-Type: application/json // Payload shape { event: "execution.complete", data: { ... }, timestamp: "2026-03-22T..." }

Retry Strategy

// Exponential backoff: 1s, 4s, 9s setTimeout(resolve, attempt * attempt * 1000) // Max attempts: 3 // Timeout per attempt: 10s // Status tracking in DB: pending | success | failed

Database Tables

webhooks

url, events[], secret, enabled

webhook_deliveries

payload, responseCode, responseBody, attempts, status

HMAC-SHA256: webhooksService.signPayload(payload, secret) gera a assinatura. O recipient pode verificar com crypto.createHmac('sha256', secret).update(payload).digest('hex').

Isolation

// Directory isolation hook // Blocks: Read, Write, Edit, Glob, // Grep, Bash outside project ALLOWED_DIR={projectPath} // Permitted exceptions: /tmp, /dev/null, /proc /usr, /bin, /etc /home

Project isolation: Cada projeto tem projectPath unico. O sistema sincroniza apenas recursos do DB para o FS. Arquivos criticos (.mcp.json, settings.json) sao regravados a cada execucao. Recursos globales sao mesclados com os do step.
