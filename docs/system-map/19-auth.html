// api-key-enforcement.ts export async function apiKeyEnforcement(request, reply) { const keyInfo = request.apiKeyInfo if (!keyInfo) return // Nao e API key, pula // 1. IP Whitelist if (keyInfo.ipWhitelist.length > 0) { if (!keyInfo.ipWhitelist.includes(request.ip)) throw new ForbiddenError(`IP ${request.ip} not in whitelist`) } // 2. Scope Check const routeScope = deriveRouteScope(request.method, request.url) const hasScope = keyInfo.scopes.some(s => matchScope(s, routeScope)) if (!hasScope) throw new ForbiddenError(`Lacks scope for ${routeScope}`) // 3. Rate Limit (Redis per-key counter, TTL 60s) const current = await redis.incr(`ratelimit:apikey:${keyInfo.id}`) if (current > keyInfo.rateLimit) throw new ForbiddenError('Rate limit exceeded') }

Sistema de Scopes
📄
Formato

Scopes seguem o padrao resource:action

workflows:read Ler workflows
executions:write Criar execucoes
admin:delete Deletar como admin
*:* Acesso total (wildcard)
💡
Wildcards

Wildcards permitem acesso amplo com uma unica scope.

* — Acesso total a tudo
workflows:* — Todas as acoes em workflows
*:read — Leitura em todos os recursos
*:delete — Delete em todos os recursos

// matchScope() — Logica de correspondencia function matchScope(scope: string, route: string): boolean { if (scope === '*' || scope === '*:*') return true const [scopeRes, scopeAct] = scope.split(':') const [routeRes, routeAct] = route.split(':') if (scopeRes === '*') return true if (scopeRes !== routeRes) return false if (scopeAct === '*') return true return scopeAct === routeAct } // deriveRouteScope() — Extrai scope da URL // GET /api/workflows/abc123 → "workflows:read" // POST /api/executions → "executions:write" // DELETE /api/api-keys/id → "api_keys:delete"
🔁
Ciclo de Vida da API Key
1
Geracao
POST /api-keys com { name }. Gera 32 bytes aleatorios via crypto.randomBytes(32), codifica em base64url com prefixo exk_. Armazena: hash SHA256 + prefix (primeiros 12 chars). A chave raw so e retornada UMA VEZ na response de criacao.
2
Uso
Cada request com Bearer exk_... incrementa usageCount e atualiza lastUsedAt (fire-and-forget, nao bloqueia a request).
3
Listagem
GET /api-keys retorna todas as keys do usuario (nao revogadas). Mostra prefix, name, lastUsedAt, createdAt. Nunca retorna a key raw ou o hash.
4
Revogacao
DELETE /api-keys/:id seta revoked = true. A key nao e deletada do banco (soft delete para auditoria). Requests subsequentes com essa key retornam 401.
5
Expiracao
Campo expiresAt opcional. Se definido e a data passou, o middleware rejeita com 401 "API key expired". Keys sem expiresAt nao expiram automaticamente.

model ApiKey { id String @id @default(uuid()) name String keyHash String @unique @map("key_hash") prefix String // primeiros 12 chars (exk_abc1...) scopes Json @default("[]") // ["workflows:read", "executions:*"] rateLimit Int? @map("rate_limit") // requests por minuto usageCount Int @default(0) ipWhitelist Json @default("[]") // ["192.168.1.1", "10.0.0.0/8"] lastUsedAt DateTime? expiresAt DateTime? revoked Boolean @default(false) createdAt DateTime @default(now()) userId String @map("user_id") user User @relation(fields: [userId], references: [id], onDelete: Cascade) @@index([userId]) @@index([keyHash]) @@map("api_keys") }

IP Whitelist & Rate Limiting
🌍
IP Whitelist

Lista opcional de IPs permitidos por API key. Se definida, requests de outros IPs sao rejeitados com 403 Forbidden.

// Verificacao de IP if (keyInfo.ipWhitelist.length > 0) { if (!keyInfo.ipWhitelist.includes(request.ip)) throw new ForbiddenError( `IP ${request.ip} not in API key whitelist` ) }

Armazenado como JSON array no campo ipWhitelist da ApiKey.

📈
Rate Limiting

Dois niveis de rate limiting:

Global

100 req/min por user/IP (@fastify/rate-limit)

Auth endpoints

5 req/min em /auth/login e /auth/register

Por API Key

Configuravel por key via campo rateLimit. Counter Redis com TTL 60s.

Execucoes

5 req/min para criar execucoes

// middlewares/rbac.ts export function requireRole(...allowedRoles: string[]) { return async (request, reply) => { const userId = await request.getCurrentUserId() const user = await prisma.user.findUnique({ where: { id: userId }, select: { role: true }, }) if (!user || !allowedRoles.includes(user.role)) throw new ForbiddenError('You do not have permission') } } // Uso nas rotas: server.get('/admin/users', { preHandler: [requireRole('admin')] }, handler) server.post('/workflows', { preHandler: [requireRole('admin', 'developer')] }, handler)

Endpoints de Autenticacao
Metodo Rota Descricao
POST /auth/register Registra novo usuario. Body: { email, password, name? }. Retorna { token, user }. Rate limit: 5/min.
POST /auth/login Login com email/password. Retorna { token, user }. Rate limit: 5/min.
GET /auth/me Retorna perfil do usuario autenticado. Requer Bearer token. Inclui hasGithub.
GET /auth/budget Resumo de budget de tokens: dailyUsage, dailyLimit, monthlyUsage, monthlyLimit, percentuais.
Metodo Rota Descricao
POST /api-keys Gera nova API key. Body: { name }. Retorna a key raw (unica vez) + id, prefix, createdAt.
GET /api-keys Lista API keys do usuario (nao revogadas). Retorna prefix, name, lastUsedAt, createdAt, revoked.
DELETE /api-keys/:id Revoga API key (soft delete: revoked = true). Retorna 204 No Content.
