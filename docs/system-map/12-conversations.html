Visao Geral
Conversations sao instancias de execucao ativas associadas a um Workflow. Quando um usuario inicia uma nova conversa, o sistema cria um registro Conversation vinculado a um workflow e a uma pasta de projeto no disco.

Cada conversa contem:
💬
Messages
Mensagens user/assistant/system com metadados, contagem de tokens, anexos e selecao de contexto.
⚡
Execution States
Estado da execucao atual: running, paused, completed, failed. Rastreio de step atual e tokens.
📊
Traces & Logs
Cada chamada ao Claude CLI gera um trace com stdout/stderr, duracao, exit code e parsed events.

Fork (Branch)
Cria uma nova conversa a partir de uma existente, copiando workflow e projectPath. O campo parentId referencia a conversa original. Util para testar abordagens alternativas sem perder o historico.
POST /conversations/:id/clone // Response: nova conversa com parentId = :id { "id": "uuid-novo", "workflowId": "mesmo-workflow", "parentId": "uuid-original" }
📦
Archive
Conversas podem ser arquivadas via o campo isArchived. Conversas arquivadas nao aparecem nas listagens padrao mas continuam acessiveis para consulta e export.
PATCH /conversations/:id { "isArchived": true }

Controles de Execucao

⏸ Pause / Resume ▼

A execucao pausa automaticamente quando o Claude CLI emite um evento ask_user. O frontend exibe a pergunta e o usuario pode responder. O endpoint POST /messages/stream detecta a pausa e retoma automaticamente com a resposta do usuario.

🛑 Cancel ▼

POST /conversations/:id/cancel - Cancela a execucao ativa. O processo Claude CLI e terminado via signal e o SSE emite cancelled.

💬 Interrupt ▼

POST /conversations/:id/interrupt - Envia mensagem durante execucao ativa. Util para redirecionar o Claude sem cancelar. O texto e injetado via stdin no processo.

👀 Watch ▼

GET /conversations/:id/watch - SSE read-only para observar uma execucao ativa sem iniciar uma nova. Util para multiplas abas ou monitoramento.

Step Navigation (step_by_step mode)

POST /:id/advance-step Avanca para o proximo step
POST /:id/go-back-step Volta para o step anterior
POST /:id/jump-to-step Pula direto para qualquer step (body: { stepId })

Cada conversa e vinculada a um projectPath no disco. O Claude CLI opera dentro dessa pasta, lendo e escrevendo arquivos do projeto.

GET /folders

GET /folders // Lista pastas de projeto do usuario logado // Response: [ { "name": "meu-saas", "path": "/workspace/temp-orquestrador/users/abc123/projetos/meu-saas", "conversationsCount": 3 }, { "name": "loja-virtual", "path": "/workspace/temp-orquestrador/users/abc123/projetos/loja-virtual", "conversationsCount": 1 } ]

Fork & Clone

Como funciona

O clone cria uma nova conversa com:
  • Mesmo workflowId da conversa original
  • Mesmo projectPath
  • parentId apontando para a conversa original
  • Titulo com sufixo "(clone)"
  • Historico de mensagens NAO e copiado (comeca limpo)
  • currentStepId resetado para o primeiro step

Diagrama de Relacionamento

📝 Conversa Original (parent)
🍴 Clone 1 (parentId = original)
🍴 Clone 2 (parentId = original)

O campo children no model Conversation permite navegar de parent para clones via @relation("ConversationParent").

Memory Management
📝
StepMemory
Cada par (conversationId, stepId) pode ter um StepMemory associado. Este registro contem um resumo compactado do contexto acumulado da conversa ate aquele step.

Quando a conversa fica longa demais para o contexto do Claude, o sistema compacta as mensagens antigas em um resumo e salva como StepMemory. Nas proximas execucoes, o StepMemory e injetado no system prompt para manter a continuidade.
🔄
Compactacao de Contexto
O fluxo de compactacao funciona assim:
  1. Mensagens se acumulam durante a conversa
  2. Quando o total de tokens excede o limite, o MemoryManager e acionado
  3. Mensagens antigas sao resumidas via Claude API
  4. O resumo e salvo como StepMemory
  5. Mensagens antigas marcadas como selectedForContext = false
  6. SSE emite evento context_reset

Sessions (Claude CLI)

ConversationSession

Cada combinacao (conversationId, stepId) tem uma sessao Claude CLI associada. O claudeSessionId permite retomar o contexto do Claude entre execucoes.

Quando o usuario envia uma nova mensagem no mesmo step, o sistema passa o --resume flag com o token da sessao anterior, mantendo todo o contexto (arquivos lidos, historico de ferramentas, etc.).

O endpoint DELETE /conversations/:id/sessions/:stepId permite resetar a sessao de um step especifico, forcando o Claude a comecar do zero naquele step.

Fluxo de Resume

1 Mensagem chega no step X
2 SessionManager busca sessao para (conv, stepX)
3 Se existe: --resume {claudeSessionId}
4 Se nao: nova sessao, salva o ID no banco
5 Trace registra resumeTokenOut para proxima vez

AI Suggestions

GET /conversations/:id/suggestions

Gera sugestoes de proximos passos baseadas no historico da conversa usando Claude Haiku. Analisa:
  • System prompt do step atual e do workflow inteiro
  • Ultimas 20 mensagens da conversa
  • Ferramentas usadas na ultima execucao
  • Objetivo original do usuario (primeira mensagem)

Retorna exatamente 4 sugestoes com text (instrucao para colar no chat) e description (justificativa).
GET /conversations/:id/suggestions // Response: { "suggestions": [ { "text": "Crie os testes de integracao para a API de autenticacao", "description": "API criada, proximo passo e validar com testes" }, { "text": "Implemente o middleware de rate limiting", "description": "Seguranca essencial antes de ir para producao" } ] }
