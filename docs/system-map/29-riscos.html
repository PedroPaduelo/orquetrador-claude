🛡️ Riscos de Seguranca

Alto SEC-001

Token/Secret Exposure

stdout/stderr em traces de execucao podem conter secrets, tokens de API e credenciais. Dados sensiveis ficam visiveis nos logs e na interface de monitoramento.

Impacto
Vazamento de credenciais, acesso nao autorizado a servicos externos
Mitigacao
  • → Sanitizar output de traces antes de armazenar
  • → Encriptar campos sensiveis com ENCRYPTION_KEY
  • → Mascarar tokens nos logs (ex: sk-...xxxx)
Alto SEC-002

Sandbox Escape

Claude CLI tem acesso ao sistema mesmo dentro do sandbox. O usuario claude-sandbox pode acessar areas nao previstas do filesystem.

Impacto
Execucao de comandos arbitrarios, leitura de arquivos do sistema
Mitigacao
  • → Auditar permissoes do sandbox periodicamente
  • → Limitar acesso ao filesystem (chroot ou namespaces)
  • → Implementar whitelist de comandos permitidos
Medio SEC-003

API Key Leakage

Chaves de API armazenadas em memoria e variaveis de ambiente podem ser expostas via dumps de memoria, logs de debug ou endpoints de diagnostico.

Impacto
Uso indevido de API keys, custos inesperados
Mitigacao
  • → Usar ENCRYPTION_KEY para encriptacao at-rest
  • → Rotacionar keys periodicamente
  • → Nunca logar valores de env vars sensiveis
Medio SEC-004

Rate Limiting Desabilitado

O rate limit global esta OFF em producao. A API esta vulneravel a ataques de forca bruta, DDoS e abuso de recursos.

Impacto
Indisponibilidade do servico, consumo excessivo de recursos
Mitigacao
  • → Habilitar @fastify/rate-limit em producao
  • → Configurar limites por rota (auth: mais restrito)
  • → Adicionar rate limit por IP e por usuario
Medio SEC-005

GitHub Token Storage

Tokens do GitHub armazenados como texto plano no banco de dados. Se o banco for comprometido, todos os tokens ficam expostos.

Impacto
Acesso nao autorizado a repositorios, push malicioso
Mitigacao
  • → Encriptar com ENCRYPTION_KEY (AES-256-GCM)
  • → Usar tokens com escopo minimo necessario
  • → Implementar rotacao automatica
Baixo SEC-006

CORS Permissivo

Configuracao CORS aceita multiplas origens. Embora nao seja wildcard, a lista e ampla e pode permitir origens nao desejadas.

Impacto
Potencial CSRF se origem maliciosa estiver na lista
Mitigacao
  • → Restringir origens a dominios conhecidos
  • → Usar validacao dinamica de origin
Baixo SEC-007

Webhook Secrets em Texto Plano

Secrets de webhooks armazenados sem encriptacao no banco. Comprometimento do banco expoe secrets de validacao de payloads.

Impacto
Falsificacao de payloads de webhook
Mitigacao
  • → Encriptar secrets com ENCRYPTION_KEY
  • → Implementar rotacao de secrets
