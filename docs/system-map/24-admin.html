Visao Geral

O modulo Admin centraliza funcionalidades administrativas do Orquetrador Claude: gerenciamento de usuarios (criar, editar role, deletar), visualizacao de audit logs e configuracao do sistema. Todas as rotas sao protegidas por middleware de verificacao de role admin.

Audit Log System

Toda acao relevante no sistema gera um registro de auditoria. O modelo AuditLog captura quem fez, o que fez, em qual recurso, e o diff (before/after) da alteracao.

AuditLog Model

CampoTipoDescricao
id String CUID, primary key
userId String FK para User - quem executou a acao
action String create | update | delete | execute | login
resourceType ResourceType Enum: workflow, skill, agent, plugin, etc.
resourceId String? ID do recurso afetado (opcional para login)
resourceName String? Nome legivel do recurso
diff Json? { before: {...}, after: {...} }
metadata Json? Dados adicionais (IP, user-agent, etc.)
ipAddress String? Endereco IP da requisicao
createdAt DateTime Timestamp da acao

Acoes Rastreadas

create update delete execute login

Cada acao gera um AuditLog automaticamente via middleware ou chamada direta no service layer.

Resource Types

workflow skill agent plugin hook rule mcpServer conversation execution webhook trigger user tag gitSource

Todos os valores do enum ResourceType do Prisma.

🔒
Seguranca

Role Check Middleware

Todas as rotas /admin/* passam pelo middleware requireRole('admin') que:

1. Verifica JWT valido (via authenticate)
2. Extrai user.role do token
3. Compara com role requerido
4. Retorna 403 Forbidden se nao autorizado

Protecoes Adicionais

• Admin nao pode deletar a si mesmo
• Ultimo admin nao pode ter role rebaixado
• Toda alteracao de role gera AuditLog
• Delete gera AuditLog antes de remover
• Rate limit em rotas admin (mais restritivo)
• IP registrado em todas as acoes
