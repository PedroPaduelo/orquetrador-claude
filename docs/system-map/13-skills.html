🧠 O que sao Skills

📝
Definicao
Skills sao modulos de inteligencia customizados que podem ser anexados a steps de workflow ou agents. Sao escritos em markdown/codigo e definem instrucoes especificas, ferramentas permitidas (tool whitelist) e comportamentos que o Claude deve seguir ao executar um step.
⚙️
Funcionalidades
  • CRUD Criar, editar, excluir e listar skills
  • Import Importar de repositorios GitHub
  • Sync Sincronizar com repos remotos
  • Test Testar execucao de skills
  • Attach Anexar a workflow steps e agents
  • Toggle Habilitar/desabilitar via MCP

🗄️ Modelo de Dados

📊
Skill (Prisma Model)
model Skill { // Identidade id String @id @default(uuid()) name String description String? body String // markdown/code // Metadata frontmatter Json? allowedTools String[] model String? enabled Boolean @default(true) isGlobal Boolean @default(false) source SkillSource // GitHub tracking repoUrl String? repoOwner String? repoName String? repoBranch String? repoPath String? fileManifest Json? lastSyncedAt DateTime? // Relations userId String pluginId String? // Joins workflowSteps WorkflowStepSkill[] agents AgentSkill[] }

Campos em detalhe

  • id String (UUID) Chave primaria
  • name String Unico por usuario
  • description String? Descricao curta
  • body String Conteudo markdown/codigo da skill
  • frontmatter Json? Metadata JSON (parsed do body)
  • allowedTools String[] Whitelist de ferramentas permitidas
  • model String? Modelo AI especifico (opcional)
  • enabled Boolean Skill ativa (default: true)
  • isGlobal Boolean Disponivel para todos os users
  • source SkillSource manual | imported
  • repoUrl String? URL completa do repo GitHub
  • repoOwner String? Owner do repo (org/user)
  • repoName String? Nome do repo
  • repoBranch String? Branch rastreada
  • repoPath String? Caminho dentro do repo
  • fileManifest Json? Lista de arquivos rastreados
  • lastSyncedAt DateTime? Ultimo sync com GitHub
  • pluginId String? Se veio de um plugin

🤖 MCP Tools (7)

Ferramentas disponiveis via MCP Server para interacao programatica com skills.

skill_list
Lista todas as skills do usuario com paginacao e filtros.
params: page?, limit?, search?, source?, enabled?
skill_get
Obtem detalhes completos de uma skill incluindo body e metadata.
params: skillId
skill_create
Cria uma nova skill com nome, body, allowedTools e configuracoes.
params: name, body, description?, allowedTools?, model?
skill_update
Atualiza campos de uma skill existente (partial update).
params: skillId, name?, body?, allowedTools?, ...
skill_delete
Remove uma skill permanentemente. Desvincula de steps e agents.
params: skillId
skill_toggle
Habilita ou desabilita uma skill sem excluir.
params: skillId, enabled
skill_import
Importa skill(s) de um repositorio GitHub (single ou bulk).
params: repoUrl, branch?, path?

📂 GitHub Integration

🔗
Capacidades
  • ✔ Importar skills de qualquer repo GitHub publico ou privado
  • ✔ Rastrear file manifests (SHA de cada arquivo)
  • ✔ Sincronizacao manual via PATCH /skills/:id/sync
  • ✔ Bulk import de todos os .md de um diretorio
  • ✔ Source tracking: manual vs imported
  • ✔ lastSyncedAt timestamp para controle
📥
Fluxo de Import

1. Informar URL do Repositorio

Usuario fornece repoUrl, branch e path (opcional) do diretorio de skills.

2. Fetch do GitHub API

Backend busca lista de arquivos .md no path especificado via GitHub Contents API.

3. Parse e Validacao

Cada arquivo e parseado: extrai frontmatter (YAML/JSON), body, allowedTools. Valida schema.

4. Criacao no Banco

Skills criadas com source: "imported", repoUrl, repoOwner, repoName, repoBranch preenchidos.

5. File Manifest Salvo

Array de {path, sha} gravado para detectar mudancas no proximo sync.

🔄 Uso em Workflows

🎯
Como Skills funcionam em Workflow Steps
Skills sao anexadas a steps de workflow via a join table WorkflowStepSkill. Quando um step e executado, o motor de execucao injeta o body da skill no system prompt do Claude e restringe as ferramentas disponiveis ao allowedTools definido na skill.

Injecao no System Prompt
// No motor de execucao (execution-engine) // 1. Busca skills do step const stepSkills = await prisma.workflowStepSkill .findMany({ where: { workflowStepId: step.id }, include: { skill: true } }) // 2. Filtra apenas skills habilitadas const activeSkills = stepSkills .filter(ss => ss.skill.enabled) // 3. Monta system prompt com skills const systemPrompt = [ step.prompt, ...activeSkills.map(ss => `\n--- SKILL: ${ss.skill.name} ---\n` + ss.skill.body ) ].join('\n')
🔒
Restricao de Tools
// allowedTools limita o que o Claude pode usar // Exemplo de skill com tools restritas: { "name": "code-review", "allowedTools": [ "Read", "Grep", "Glob" ] // Bash, Write, Edit: BLOQUEADOS // Claude so pode LER, nao modificar } // Exemplo sem restricao: { "name": "full-dev", "allowedTools": [] // Array vazio = todas permitidas } // O motor aplica a intersecao: // finalTools = skill.allowedTools.length > 0 // ? intersect(defaultTools, skill.allowedTools) // : defaultTools

🤝 Relacao com Agents

🤖
AgentSkill — Vinculacao Agent ↔ Skill
Agents podem ter skills associadas via a join table AgentSkill. Quando um agent e invocado, suas skills sao combinadas com as skills do step (se houver) para formar o contexto completo de execucao.

Step skills (via WorkflowStepSkill) // Skills do step // allowedTools: intersecao de todos os niveis // Se agent tem ["Read","Bash"] e step skill tem ["Read","Grep"] // => finalTools = ["Read"] (intersecao)
